# GradQuest 발표 가이드

## 한 줄 소개

GradQuest는 흩어진 성균관대학교 졸업요건 문서를 AI가 구조화하고, 학생별 충족 상태와 다음 행동을 보여주는 졸업 내비게이터다.

## 5분 발표 구성

### 1. 문제 공감 - 35초

“졸업을 앞둔 학생은 학점만 확인하면 될까요? 입학연도, 전공 형태, 3품, 국제어수업, 학과별 졸업평가까지 직접 찾아 비교해야 합니다. 정보가 틀린 것이 아니라 흩어져 있다는 것이 문제입니다.”

화면에는 학칙, 학사 안내, 학과 공지, PDF 표가 흩어진 모습을 보여준다.

### 2. 해결책 - 35초

“GradQuest는 학교 문서를 읽어 규칙으로 바꾸고, 학생 프로필과 이수 현황에 대조합니다. 학생은 충족률, 부족 요건, 다음 행동을 한 화면에서 확인합니다.”

`문서 → 구조화된 규칙 → 개인별 진단 → 행동 추천` 흐름을 한 장에 표시한다.

### 3. 실제 데모 - 2분 10초

1. 로그인 화면에서 `글로벌경영학과 데모`를 선택한다.
2. 대시보드에서 전체 진행률과 부족한 전공·국제어·글로벌 활동을 짚는다.
3. 졸업요건 상세에서 이수 학점을 수정하고 진행률이 즉시 바뀌는 것을 보여준다.
4. AI 학사 도우미에서 “지금 가장 먼저 해야 할 일은?”을 질문한다.
5. 소프트웨어학과 조기졸업 데모로 전환해 `DASF003·DASF004` 학과 지정 DS 예외가 자동 적용되고, 6~7학기·평점·전공 66학점·연구 및 인턴십 요건이 함께 계산되는 장면을 보여준다.

문서 업로드는 실제 Upstage API가 연결된 배포본에서 한 번만 시연한다. 네트워크 문제가 생기면 샘플 데이터로 동일한 결과를 보여주는 순서를 준비한다.

### 4. Upstage 기술 활용 - 45초

“Document Parse는 복잡한 PDF 표와 공지를 읽고, Solar는 추출 내용을 규칙 후보로 바꾸며 학생의 질문에 자료 중심으로 답합니다. 중요한 규칙은 사람이 승인한 뒤 서비스에 반영합니다.”

API 호출 자체보다 문서가 실제 학생 행동으로 이어지는 제품 흐름을 강조한다.

### 5. 차별점과 확장성 - 35초

“기존 서비스가 학점 체크에 집중했다면 GradQuest는 전공 형태, 졸업평가, 조기졸업까지 설명하고 다음 행동을 제시합니다. 먼저 글로벌경영학과를 정확하게 검증한 뒤 전 학과와 비교과 추천으로 확장할 수 있습니다.”

### 6. 마무리 - 20초

“학생이 규정을 읽는 데 시간을 쓰지 않고, 졸업을 위해 무엇을 해야 하는지 이해하도록 만드는 것. 그것이 GradQuest가 해결하려는 문제입니다.”

## 슬라이드 구성

| 장 | 제목 | 핵심 시각 요소 |
|---|---|---|
| 1 | 졸업요건, 왜 이렇게 찾기 어려울까? | 흩어진 문서 4종 |
| 2 | 한 학생에게도 기준이 여러 개다 | 글로벌경영학과 페르소나 |
| 3 | GradQuest | 제품 한 줄 소개와 흐름 |
| 4 | AI가 문서를 규칙으로 바꾸는 과정 | Parse–Solar–검수 다이어그램 |
| 5 | 실시간 데모 | 대시보드 전체 화면 |
| 6 | 기존 서비스와 무엇이 다른가 | 4열 비교표 |
| 7 | 구현 구조와 신뢰성 | 프론트–API–데이터 구조 |
| 8 | 학교 전체로 확장 | 학과·비교과 로드맵 |
| 9 | Graduation, made navigable | 로고와 한 문장 |

## 발표자료 디자인

- 16:9, 흰 배경을 기본으로 하고 짙은 녹색을 제목·강조에만 사용한다.
- 한 슬라이드에는 한 문장과 한 시각 자료만 둔다.
- 본문 24pt 이상, 제목 34pt 이상을 유지한다.
- 제품 화면은 장식 프레임보다 실제 크롭 이미지를 크게 배치한다.
- 상태 색상은 녹색·금색·빨간색을 사용하되 텍스트 라벨도 함께 표시한다.
- 기술 다이어그램은 `입력 문서 → Parse → 규칙 DB → 진단 → Solar Q&A`의 한 방향으로 단순화한다.

## 예상 질문

**AI가 틀린 졸업요건을 안내하면 어떻게 하나요?**  
원문 위치와 적용 연도를 함께 저장하고, AI 추출값은 관리자 검수 후 규칙으로 승인한다. 사용자 화면에도 공식 출처와 참고용 진단임을 표시한다.

**GLS와 무엇이 다른가요?**  
GLS가 행정 기록과 최종 판정의 기준이라면 GradQuest는 여러 규칙을 학생 언어로 설명하고 부족 항목의 우선순위를 제시하는 탐색·준비 도구다.

**모든 학과를 어떻게 관리하나요?**  
처음부터 전 학과를 완벽히 지원하지 않고 글로벌경영학과를 검증 대상으로 삼는다. 이후 공통 규칙과 학과별 규칙을 분리해 학과 단위로 확장한다.

**개인정보는 안전한가요?**  
현재 MVP는 샘플 데이터 중심이다. 운영 버전에서는 학교 인증, 서버 저장 암호화, 최소 수집, 성적표 비식별 처리와 삭제 정책이 필요하다.

## 리허설 체크리스트

- 발표용 배포 URL과 로컬 백업 URL을 모두 준비한다.
- 글로벌경영학과 데모 계정이 첫 화면에 노출되는지 확인한다.
- 소프트웨어학과 데모에서 `학과 지정 예외` 배지와 두 DS 과목이 보이는지 확인한다.
- 학점 수정 후 진행률이 바뀌는 장면을 20초 안에 끝낸다.
- AI 질문은 미리 정한 한 문장으로 고정한다.
- Document Parse 실패 시 사용할 샘플 결과 화면을 준비한다.
- 발표자는 4분 30초에 마치고 30초를 돌발 상황에 남긴다.
- 팀원 세 명이 문제·제품, 기술·데모, 차별점·마무리를 나누어 말한다.
