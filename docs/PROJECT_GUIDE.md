# GradQuest 프로젝트 가이드

## 1. 문제 정의

성균관대학교의 졸업요건은 학사제도 안내, 학칙, 전공별 공지, PDF 기준표에 흩어져 있다. 같은 학과라도 입학연도와 단일전공·복수전공·연계전공 여부에 따라 기준이 달라 학생이 자신의 상태를 직접 대조하기 어렵다.

GradQuest는 학생 프로필과 이수 현황을 구조화된 졸업요건에 대조하여 다음 세 가지를 한 화면에서 제공한다.

1. 현재 충족률
2. 부족한 요건과 부족 학점
3. 다음으로 해야 할 행동

결과는 참고용 진단이며 최종 판정은 GLS, 최신 학사제도, 소속 학과 사무실에서 확인하도록 안내한다.

## 2. 핵심 사용자

- 1순위: 글로벌경영학과 단일전공 학생
- 화학공학부 + 반도체소재부품장비패키징 융합트랙 학생
- 문헌정보학과 + 경제학과 복수전공 학생
- 소프트웨어학과 조기졸업 준비 학생: 2020학번 지정 DS 과목과 연구·인턴십 졸업평가 예외 검증

## 3. 벤치마킹 요약

| 서비스·자료 | 참고할 점 | GradQuest 적용 | 차별화 방향 |
|---|---|---|---|
| Smunity | 학교별 졸업요건을 체크리스트로 단순화 | 요건별 상태와 부족 수치 표시 | 전공 형태와 입학연도를 함께 계산 |
| Graduation-Okay | 과목 데이터를 이용한 자동 판정 구조 | 계산 로직을 데이터와 분리 | 성균관대 졸업평가·3품·국제어수업까지 확장 |
| PleaseGraduate | 이수 내역 중심의 진행률 시각화 | 진행률 링과 영역별 상태 | 다음 행동 우선순위와 AI 설명 제공 |
| 개인 Notion 학점표 | 사용자가 직접 수정하기 쉬움 | 이수 학점 즉시 수정·저장 | 공식 문서 파싱으로 입력 부담 감소 |

핵심 차별점은 단순 학점 계산이 아니라 `문서 수집 → 규칙 구조화 → 개인별 판정 → 인정내역과 다음 행동 제시`를 하나의 흐름으로 구현하는 것이다.

## 4. 정보 구조

```text
로그인/회원가입
  └─ 학생 프로필 설정
      ├─ 대시보드
      │   ├─ 영역별 진행률
      │   ├─ 부족 요건
      │   └─ 우선 행동
      ├─ 졸업요건 상세
      │   ├─ 학점·교양·전공
      │   ├─ 3품·졸업평가
      │   └─ 공식 출처
      ├─ 조기졸업 계산
      ├─ AI 학사 도우미
      └─ 팀/프로그램 소개
```

## 5. 데이터 스키마

```js
{
  student: {
    admissionYear,
    campus,
    college,
    department,
    degreeType,
    secondaryProgram,
    currentSemester,
    gpa,
    earlyGraduation
  },
  requirements: {
    totalCredits: { completed, required },
    coreGeneral: { completed, required },
    balancedGeneral: { completed, required },
    dsEducation: { completed, required, exception, exceptionLabel, detail, courseCodes },
    primaryMajor: { completed, required, detail },
    secondaryMajor: { completed, required, detail },
    internationalTotal: { completed, required },
    internationalMajor: { completed, required },
    poom: [{ id, label, completed }],
    graduationEvaluation: {
      label,
      description,
      checklist: [{ label, completed }]
    }
  },
  evidence: {
    sourceUrl,
    sourceTitle,
    effectiveYear,
    lastVerifiedAt
  }
}
```

운영 단계에서는 각 규칙에 `입학연도 범위`, `학과`, `전공 형태`, `예외 조건`, `원문 위치`, `검수자`를 함께 저장해야 한다. AI가 추출한 값과 사람이 검수한 값을 분리하면 변경된 학사제도도 안전하게 관리할 수 있다.

## 6. 판정 방식

1. 학생의 입학연도·학과·전공 형태에 맞는 규칙 세트를 선택한다.
2. 각 요건의 `completed / required`를 계산한다.
3. 학점 외 조건은 체크리스트의 완료 여부로 판정한다.
4. 미충족 항목을 마감 위험과 부족량 기준으로 정렬한다.
5. 공식 출처와 주의 문구를 결과에 함께 표시한다.

조기졸업은 6~7학기, 평점 4.00 이상, 전체 졸업요건 100% 충족을 별도 계산한다. 학부-대학원 연계과정의 조기졸업 기준과 혼동하지 않도록 화면에서 구분한다.

페르소나 4는 2020학번 소프트웨어학과 단일전공 조기졸업으로 설정한다. 일반 DS 기본과목 대신 `DASF003 공학컴퓨터프로그래밍`과 `DASF004 프로그래밍기초와실습` 6학점을 적용하는 학과 지정 예외를 표시하고, 전공 66학점 및 연구논문·연구작품 택1과 인턴십 졸업평가를 별도로 추적한다.

## 7. 문서 AI 활용

- Document Parse: 학사 안내 PDF·학과 공지에서 표, 제목, 조건, 각주 추출
- Solar: 추출 결과를 스키마 후보로 변환하고 학생 질문에 자료 중심으로 답변
- 사람 검수: 입학연도 예외, 중복인정, 졸업평가처럼 오판 위험이 큰 조건 승인

MVP의 `/api/parse`와 `/api/chat`은 이 흐름을 보여주는 서버리스 연결점이다. API 키는 브라우저에 두지 않고 Vercel 환경변수로만 설정한다.

## 8. 화면 원칙

- 성균관대 이미지를 연상시키는 짙은 녹색, 흰색, 차가운 회색을 기본으로 사용한다.
- 첫 화면에서 전체 진행률과 가장 급한 부족 요건을 동시에 보여준다.
- 합격·주의·미충족 상태를 색상뿐 아니라 문구와 아이콘으로 구분한다.
- 모바일에서는 사이드바를 상단 메뉴로 바꾸고 모든 카드가 한 열로 쌓이게 한다.
- 결과마다 공식 출처와 `최종 확인 필요` 문구를 유지한다.

## 9. 데이터 확장 우선순위

1. 글로벌경영학과 최근 5개 입학연도 기준 검수
2. 공통 졸업요건과 3품·국제어수업 규칙 검수
3. 나머지 세 페르소나의 학과별 졸업평가 원문 연결
4. GLS 성적표 또는 수강내역 파일의 개인정보 제거·파싱 실험
5. 자연과학캠퍼스 전 학과와 비교과 프로그램으로 확장
